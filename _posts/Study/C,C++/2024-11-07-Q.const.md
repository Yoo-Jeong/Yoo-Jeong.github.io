Q. const에 대해  
A. const는 이 코드는 값을 바꾸지 않겠다는 의도 표현이며, 컴파일 타임 안전성과 코드 신뢰도를 높여줍니다.   
1. 일반 변수의 const
    - 값 변경 불가
    - 반드시 초기화 필요

2. 포인터와 const
```cpp
const int* p1;   // 가리키는 값이 const  
int* const p2;   // 포인터 자체가 const  
const int* const p3; // 둘 다 const  
``` 
    - const int* → *p1 변경 불가, 주소 변경 가능  
    - int* const → 주소 변경 불가, *p2 변경 가능
    - 오른쪽부터 읽기가 가장 안전한 해석법  

3. 함수 매개변수의 const
    - 함수 내부에서 값 변경 방지
    - 불빌요한 복사 방지(const reference)

4. 멤버 함수의 const
    - 함수 내부에서 멤버 변수 변경 불가
    - const 객체도 호출 가능
    - getter 함수 

5. 반환값의 const
    - 기본 타입에서는 의미 거의 없음
    - 객체/참조 반환 시 실수 방지용으로 사용
